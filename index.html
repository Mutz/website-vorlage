Markup Guide

In diesem Dokument werden Referenzen zu Elementen und Attributen gemacht. Es ist wichtig den Unterschied zu verstehen, also betrachten Sie diesen Code:

<a href="index.php">Home</a>

Das Base-HTML-Element ist das a in diesem Fall. Das Attribut ist irgendeine zusätzliche Information über das Element innerhalb der ursprünglichen < >-Paare oder der href-Wert in diesem Fall.

Um eine Beispielbenutzung für irgendein Objekt in diesem Dokument, schauen Sie sich bitte den Quellcode an.

Überschrift-Beispiele

Die zweite Überschrift oben ist ein h2-Element, welches für jede Form einer wichtigen Seitenüberschrift benutzt werden kann. Mehr als ein Element pro Seite kann benutzt werden. Versuchen Sie ein h2 einzusetzen, außer Sie benötigen eine Überschrift geringerer Wichtigkeit, oder um eine Unterüberschrift für ein bestehendes h2-Element zu benutzen. Jede Überschriftenebene kann Links enthalten, wie im Beispiel oben zu sehen.

Überschrift der 3. Ebene

Die Überschrift oben ist ein h3-Element, das für jede Art von Seiten-Ebene benutzt werden kann, die unter die h2-Überschrift fällt. Darf mehr als einmal pro Seite benutzt werden.

Überschrift der 4. Ebene

Für alle Überschriften unter der 3. Ebene, folge den Richtlinien oben. Benutze nur eine niedrigere Überschriften-Ebene, wenn nötig.

Überschrift der 5. Ebene

Für alle Überschriften unter der 4. Ebene, folge den Richtlinien oben. Benutze nur eine niedrigere Überschriften-Ebene, wenn nötig.

Überschrift der 6. Ebene

Für alle Überschriften unter der 5. Ebene, folge den Richtlinien oben. Benutze nur eine niedrigere Überschriften-Ebene, wenn nötig.

Absätze

Alle Absätze werden in p-Tags einfasst. Zusätzlich dazu können Absätze auch innerhalb eines blockquote-Elements benutzt werden, wenn das pElement ein Zitat ist. Früher wurde das blockquote-Element benutzt, um Einzüge zu erzwingen, aber dies wird jetzt mit CSS gemacht. Benutze blockquote für Zitatblöcke. Hier ist ein Beispiel einer korrekten Benutzung.:

Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Cras euismod fringilla arcu. Integer posuere. Aliquam ipsum. Donec eget massa ac orci tempus euismod. Donec quis neque nec neque consequat sollicitudin. Donec commodo tempor nulla. Suspendisse venenatis. Ut ut leo. Nunc placerat urna at libero. Nunc suscipit lacus.

lipsum.com

Zusätzlich können Sie, wenn gewünscht, eine Quellenangabe hinzufügen, wie im obigen Beispiel (Die korrekte Methode hängt das cite-Attribute direkt an das blockquote-Element an, aber da dies bislang kein Browser unterstützt, ist es nützlich die Quelle auch noch als zusätzliches, alleinstehendes cite-Element anzufügen).

Inline Text

Vielleicht ist ihnen aufgefallen, dass in dieser Richtline manchmal nichtproportionale Schrift eingesetzt wurde. Es gibt eine Anzahl von Inline-HTML-Elementen, die Sie innerhalb anderer Elemente einsetzen dürfen, z. B. abbr, cite, code, del, em, ins, kbd, strong und var.

abbr
Dieses Element wird für abgekürzten Text benutzt, egal ob bei Akronymen oder Abkürzungen. Im Allgemeines ist es weniger Arbeit und nützliche (genug) nur das erste Auftreten einer Abkürzung auf der Seite zu markieren und den Rest zu ignorieren. Jeder Text im title-Attribute erscheint, wenn der Benutzer die Maus über die Abkürzung bewegt. Beispiele für Abkürzungen und Benutzung: NASA, HTML und Mass.
code

Dieses Element wird für Computer-Code-Beispiele benutzt. Auf technischen Websites sehr nützlich, sonst eher selten gebraucht. Beispiel-Code und Benutzung:


function getJello() {
  echo $aDeliciousSnack;
}

Inline-Benutzung: Wie an anderer Stelle in diesem Dokument, können HTMLElemente wie em oder code als Code betrachtet werden und daher so markiert werden.

cite
Dieses Element wird benutzt, um ein Zitat oder eine Referenz für andere Informationsquellen zu definieren. Beispiel für zitierten Text und Benutzung: Mehr Informationen können in [ISO-0000] gefunden werden.
del
Dieses Element wird benutzt, um gelöschten oder zurückgezogenen Text, der noch auf der Seite verbleiben muss, zu markieren. Weil die standardmäßige Formatierung eine Durchstreichung enthält, sollte man es dem s-Element vorziehen. Das del-Element hat auch ein datetime-Attribute, das es ermöglicht, einen Zeitstempel direkt ins Element zu schreiben. Beispiel für gelöschten Text und Benutzung: Sie kaufte zwei fünf Paar Schuhe.
em
Dieses Element wird benutzt, um betonten Text zu markieren. In den meisten Fällen will man den Text kursiv stellen (unter Benutzung des HTML-Elements i oder sonstigem), wofür man das em-Element stattdessen benutzen sollte. Beispiel für betonten Text und Benutzung: Du musst die negitoro maki einfach probieren!
ins
Dieses Element wird benutzt, um einfügten Text zu markieren und ist das Gegenstück zum del-Element. Wie auch del, ins hat es ein datetime-Attribute, das es ermöglicht, einen Zeitstempel direkt ins Element zu schreiben. Beispiel für einfügten Text und Benutzung: Sie kaufte zwei fünf Paar Schuhe.
kbd
Dieses Element wird benutzt, um Text zu markieren, der vom Benutzer getippt werden soll. Hauptsächlich für Computeranweisungen eingesetzt. Beispiel für Benutzereingabetext und Benutzung: Bitte drücken Sie Enter um fortzufahren.
strong
Dieses Element wird benutzt, um stärkere Betonung als mit dem em-Element zu markieren. In den meisten Fällen, wo man fetten Text haben will (unter Benutzung des HTML-Elements b oder sonstigem), sollte man stattdessen das strong-Element verwenden. Beispiel für stark betonten Text und Benutzung: Keine Nägel in den elektischen Abzug stecken.
var
Dieses Element wird benutzt, um Variablen innerhalb von Computercode zu markieren. Nützlich für technologie-orientierte Websites, in allen anderen Fällen weniger. Beispiel Code mit Variablen und Benutzung: Füge 5 zu $result und berechne neu.

Listen

Dann gibt es da noch die Listen. ul markiert eine ungeordnete Liste (z. B. eine Liste von Punkten, die keine Nummerierung benötigen). ol markiert eine geordnete Liste und es sind zahlreiche verschiedene Nummerierungsformen durch CSS verfügbar (unter anderem 1,2,3… a,b,c… i,ii,iii… und so weiter). Jeder Listenpunkt innerhalb eines ul oder ol erfordert ein umfassendes <li>- und </li>-Tag, um individuelle Punkte innerhalb der Liste zu markieren. li steht für list item).

Zusätzlich gibt es noch dl als anderen Listentyp, was für Definitionsliste steht. Anstelle von Listenpunkten enthält eine dl dt (Definition Term) und dd (Definition description) Paare. Obwohl es "Defintionsliste" genannt wird, kann dl auch für andere Szenarios benutzt werden, wo es Eltern/Kindern-Verwandtschaften gibt.

Beispiel-Liste und Benutzung:

  • Dies ist eine ungeordnete Liste.
  • Sie hat zwei Punkte.
  1. Dies ist eine geordnete Liste.
  2. Sie hat zwei Punkte.
  3. Nein, das war gelogen, sie hat drei.
Dies ist ein Term.
Dies ist die Definition eines Terms, die beide in einem dl stehen.
Hier ist ein anderer Term.
Und dieser bekommt auch eine Definition, was diese Zeile ist.
